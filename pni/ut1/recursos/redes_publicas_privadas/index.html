    Redes públicas y privadas¶

    La diferencia entre una dirección IP pública y una dirección IP privada es que: Una dirección IP pública es concedida por uno de los cinco organismos regionales encargados de proporcionar direcciones IP, mientras los números de dirección IP privadas son números reservados que pueden utilizar los administradores de redes sin solicitar permiso a ninguna organización. Se recogen en el documento RFC 1918 (“Address Allocation for Private Internets”), publicado por la IETF.

    Las direcciones IP públicas se gestionan de manera jerárquica por varios organismos internacionales que garantizan su asignación ordenada y única en todo el mundo:

    La IANA (Internet Assigned Numbers Authority), que depende del ICANN (Internet Corporation for Assigned Names and Numbers). Es el organismo que administra globalmente los espacios de direcciones IP, los números de sistemas autónomos (AS) y otros parámetros de Internet.

    IANA no asigna directamente direcciones a usuarios finales, sino que delega bloques grandes a los registros regionales que son:

    Organismo Regional (RIR) Acrónimo Ámbito geográfico de aplicación Sede principal
    American Registry for Internet Numbers ARIN América del Norte (EE. UU., Canadá y parte del Caribe) Chantilly, Virginia (EE. UU.)
    Réseaux IP Européens Network Coordination Centre RIPE NCC Europa, Oriente Medio y partes de Asia Central Ámsterdam (Países Bajos)
    Asia-Pacific Network Information Centre APNIC Asia y región del Pacífico Brisbane (Australia)
    Latin America and Caribbean Network Information Centre LACNIC América Latina y el Caribe Montevideo (Uruguay)
    African Network Information Centre AFRINIC África Ebène (Mauricio)

    Las redes con IP públicas tienen acceso directo a Internet. Internet es una gran red de direcciones IP públicas.

    Las redes con IP privadas permiten proporcionar conectividad dentro de la misma red, pero no acceso instantaneo a Internet.

    Los routers implementan un mecanismo llamado NAT (Network Address Translation) o Traducción de Direcciones de Red para que los equipos de la red privada puedan acceder a Internet

    De forma resumida, cuando un dispositivo quiere acceder a Internet, el router con NAT cambia su dirección IP privada por la dirección IP pública del router.

    El router lleva un registro (tabla de traducción) que asocia cada conexión interna con el puerto y la IP pública usada, para poder reenviar correctamente las respuestas del servidor externo al equipo que las solicitó.

    Existen unos bloques de direcciones IP que podemos utilizar libremente para asignar direcciones de equipo en nuestra red local privada y que no son utilizables por equipos o nodos conectados a Internet. Completa en la siguiente tabla el número de redes y de Ips para cada uno de los rangos de Ips privadas que existen

    Tipo o clase Rango de direcciones Máscara de red por defecto Uso o descripción
    Clase A (privada) 10.0.0.0 – 10.255.255.255 255.0.0.0 (/8) Grandes redes privadas (empresas, ISP)
    Clase B (privada) 172.16.0.0 – 172.31.255.255 255.240.0.0 (/16) Redes privadas medianas (centros educativos, instituciones)
    Clase C (privada) 192.168.0.0 – 192.168.255.255 255.255.0.0 (/24) Redes privadas pequeñas (hogares, oficinas)
    Enlace local (APIPA) 169.254.0.0 – 169.254.255.255 255.255.0.0 (/16) Asignación automática cuando no hay servidor DHCP disponible
    Loopback (localhost) 127.0.0.0 – 127.255.255.255 255.0.0.0 (/8) Pruebas internas del sistema (la dirección más usada es 127.0.0.1)

    Las redes privadas se utilizan cuando se requiere que los equipos conectados a la misma deban comunicarse con otros equipos dentro de la red interna (una que no sea parte de Internet) pero no con Internet directamente.
